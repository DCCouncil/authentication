Welcome to the DC Council DataSeal Authentication Service

About the Service

Pursuant to the Uniform Electronic Legal Material Act, the Council of the District of Columbia will be required to authenticate certain primary legal materials. Specifically, the Council will be required to authenticate:

  1. The acts and resolutions of the Council
  2. The Statutes at Large
  3. The District of Columbia Official Code

To comply with this authentication requirement, the Council has partnered with the United States Open Data Institute to create an open-source, cost-effective data-authentication service. This service uses three methods to ensure that each document included in the service has not been altered in any way after it is published:

  1. HTTPS. A secure method ensuring that transmission of website data requested by a user has not been changed.
  2. SHA Hash Values. Using two cryptographic algorithms (SHA-256 and SHA-512), a unique value calculated based on the contents of a particular file. If the file has been altered in any way, the computed value will be different.
  3. Digital Signature. Using the OpenPGP standard, a method of "signing" a document to ensure that the document was actually sent by the signer.

How to use the service

The easiest way to use the service is to upload any file to the service and check whether the file is recognized by the system.

An alternative way, which does not require any additional transmission of data over the internet, is to compute the SHA256 hash value for a particular document and to check the list of hash values in the system.
